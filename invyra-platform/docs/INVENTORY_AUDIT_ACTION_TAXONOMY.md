# Inventory Audit Action Taxonomy — Phase 2B

## Status

CONTRACTED / NOT EMITTED BY NEW INVENTORY ROUTES YET

The generic AuditLog foundation already exists. Phase 2B defines the Inventory action taxonomy future services must use.

## Read Events

Read events may be sampled or limited later, but sensitive cross-tenant and denied reads must always be logged.

```text
INVENTORY_READINESS_VIEWED
INVENTORY_ITEMS_VIEWED
INVENTORY_SUPPLIERS_VIEWED

INVENTORY_CONFIGURATION_VIEWED

```

## Required Audit Fields

Every emitted Inventory audit event must include:

```text
organisationId
environmentName
module = INVENTORY
action
userId where available
resourceType where available
resourceId where available
deviceId where available
metadataJson where useful
```
